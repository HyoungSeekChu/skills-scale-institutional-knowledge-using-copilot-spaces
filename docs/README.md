# OctoAcme — Project Management Processes

This folder contains OctoAcme's core program management process documents. These artifacts capture how we initiate, plan, execute, release, and improve work across cross-functional teams.

Included docs
- octoacme-project-management-overview.md — high-level approach, roles, and lifecycle
- octoacme-project-initiation.md — how to validate and start new initiatives
- octoacme-project-planning.md — turning approved initiatives into actionable plans
- octoacme-execution-and-tracking.md — day-to-day execution rhythm and tracking
- octoacme-release-and-deployment.md — release types, deployment checklist, rollback playbook
- octoacme-risks-and-communication.md — risk register and stakeholder communication
- octoacme-retrospective-and-continuous-improvement.md — running retrospectives and tracking action items
- octoacme-roles-and-personas.md — role summaries and responsibilities

Purpose
- Make process guidance discoverable and versioned with the codebase
- Provide a single source of truth for onboarding, planning, and execution
- Encourage continuous improvement: propose changes via the .github issue template

How to propose changes
- Use the "Add Content to Project Management Process Docs" issue template in .github/ISSUE_TEMPLATE/ to propose new or updated content.
- Include a summary, rationale, and suggested content to speed editorial review.

Quick contact / ownership
- Project Manager (PM): document owner for execution guidance
- Product Lead (PdM): owner for prioritization and success metrics
- For urgent changes, open an issue and tag the PM and PdM.
